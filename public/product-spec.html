三大营收支柱（战略方向）▼
一切页面都是为了让用户完成以下三个核心动作而设计。三条路径并行存在，不是线性依赖。早期用户可以只接单赚零花钱（不需要AI资产）。
  • 三条并行路径
    • ① AI资产确权（押金） 优先级最高
      • 用户拥有AI资产 → 确权交押金 → 资产生效
      • 裂变锚点：邀请好友也来拥有AI资产
      • AI公司是AI资产的一种；未来可能有更多资产类型
      • 早期确权流程在探索中，放在「我的」Tab的AI区域
    • ② 生态任务的发布与接取
      • 接单（零门槛）→ 赚钱 → 流量入口
      • 发任务（需绑定AI资产）→ 从接单者升级为发布者
      • 平台抽成 0.1~0.5%
    • ③ AI交易与排行榜 需确认设计
      • AI资产可交易（二级市场）
      • 排行榜：资产收益/用户产出排名
      • 功能方向已定，具体设计待确认

任务状态机▼
  • 任务生命周期
    • draft 草稿（未提交，自动保存）
      • 提交 → recruiting
    • recruiting 招募中（accepted_count < total_quantity）
      • 有人接取 → in_progress（部分被接）
      • 全部接满 → fully_accepted
      • 超时无人接 → expired（资金退回）
    • in_progress 进行中（有人接取但未满）
    • fully_accepted 全部接满
    • completed 全部完成（所有接单结算完毕）
    • expired 已过期
    • cancelled 发布者取消（未被接取的部分退款）

AI资产确权流程（发布前置）▼
CEO要求：首次用户的AI资产确权需"丝滑无负担"。
来源：架构投票 f-vC
  • 用户点击"+"发布
    • 判断：用户是否有已确权的AI资产？
      • 有 → 展示资产选择列表
        • 选择已有资产 → 进入任务类型选择
        • 新建资产 → 进入创建流程
      • 无 → 进入快速创建引导
        • Step A · 选择AI公司类型
          • 预设选项 + 自定义输入
        • Step B · 选择行业/受众
          • 多选标签
        • Step C · "立即生成" → 自动确权
        • 确权完成 → 自动跳转任务类型选择
设计原则：整个确权流程控制在3步以内，30秒完成。用选择代替输入。没有等待审核环节（自动确权）。

信用分计算规则▼
维度权重计算方式
任务完成质量40%接单者：审核通过率 · 发布者：按时结算率
按时交付率25%在deadline前提交的比例
响应速度15%接单后首次操作的平均时间
对方评价20%1~5星评价的加权平均
信用分 = 各维度得分 × 权重之和。展示为三级徽章：🥇金牌（4.8+）/ 🥈银牌（4.5~4.8）/ 🆕新用户（无数据）。

    • executing 执行中
      • 提交成果 → submitted
      • AI失败 → ai_fallback（降级手动+补偿）
      • 超时未提交 → timeout（释放回池）
    • submitted 已提交，待审核
      • 审核通过 → approved
      • 审核驳回 → rejected（可重新提交，上限3次）
    • approved 审核通过 → 自动触发结算
      • 资金释放 → settled
    • settled 已结算（金额到账）
    • rejected 被驳回
      • 修改重提（retry_count < 3）→ submitted
      • 申诉 → appealing
      • 放弃 → abandoned
    • appealing 申诉中（平台24h处理）
